# trandsai（AI 伴侣）项目说明与部署

本文档汇 **项目梗概、技术总结、目录结构、部署与运维** 于一处。更细的架构记录见 [ARCHITECTURE.md](./ARCHITECTURE.md)；历史分步部署说明见 [DEPLOY.md](./DEPLOY.md)。**以本文件与代码、`.env.example` 为准**（部分旧文档中的技术栈或示例密码可能已变更）。

---

## 一、项目梗概

**trandsai** 是一套全栈 **AI 数字伴侣** 应用：用户可选择多语言虚拟角色，进行 **WebSocket 实时对话**、查看 **朋友圈（Moments）** 与互动、使用 **文化/知识增强** 与 **图像生成**；提供 **管理后台** 做伴侣与内容、配置与统计等管理。

- **用户端（client）**：H5/桌面浏览器 SPA，偏社交与聊天体验。  
- **管理端（admin）**：基于 Ant Design / Pro Components 的后台。  
- **服务端（server）**：FastAPI 统一提供 REST、WebSocket、静态资源托管（用户端构建产物 + 管理端构建产物 + 用户生成图片等）。

---

## 二、技术栈概览

| 层级 | 技术 |
|------|------|
| 用户前端 | React 18、TypeScript、Vite 6、Tailwind CSS 4、Radix UI、MUI、react-router、i18next |
| 管理前端 | React 19、TypeScript、Vite 8、Ant Design 5、Pro Components、react-router-dom、i18next |
| 后端 | Python 3.10+、FastAPI、Uvicorn、SQLAlchemy 2、Pydantic |
| AI / 数据 | LangChain / LangGraph、Anthropic / OpenAI 等（按 `.env` 配置）、ChromaDB、sentence-transformers（嵌入，按需下载） |
| 存储 | 默认 SQLite（`server/ai_companion.db`，路径以代码为准）；可选用 MySQL（PyMySQL）；向量与记忆目录见环境变量；图片本地目录托管 |

---

## 三、核心业务模块（后端视角）

- **`api/`**：路由聚合，例如 `auth`（鉴权）、`companions`（伴侣 CRUD、**WebSocket `/ws/chat/{companion_id}`**）、`moments`、`posts`、`culture`、`feedback`、`analytics`、`admin`。  
- **`services/`**：`agent` / `agent_utils`（对话 Agent）、`companion_manager`、`memory`（向量与记忆）、`image_generation`、`moments`、`posts`、`knowledge_base` 等。  
- **`core/`**：`config`、`database`（ORM）、`state`（全局 CompanionManager）、`i18n`。  
- **`main.py`**：应用生命周期：校验 **`ADMIN_PASSWORD`（禁止使用默认值 `admin123`）**、初始化数据库与 CompanionManager、后台导入文化知识、朋友圈定时任务与启动预热等；挂载静态目录与 SPA 回退。

## 四、目录结构（精简）

```
trandsai/
├── client/                 # 用户端 SPA（Vite）
│   ├── src/app/            # 页面、组件、路由、上下文等
│   ├── src/styles/
│   └── vite.config.ts      # 开发代理 → 后端（默认 localhost:8000）
├── admin/                  # 管理后台（Vite + React）
│   ├── src/
│   └── vite.config.ts      # 端口 5174，代理 /api、/data
├── server/                 # FastAPI 后端
│   ├── main.py             # 入口：静态托管、路由、SPA fallback
│   ├── api/                # 各业务路由
│   ├── services/           # 业务与 Agent 实现
│   ├── core/               # 配置、DB、状态、i18n
│   ├── scripts/            # 批处理、数据修复、种子脚本等
│   └── .env                # 后端实际读取的环境变量（需自行配置）
├── data/                   # 运行时数据（图片、记忆等，宜加入备份与 .gitignore 策略）
├── memory/                 # 知识库等 Chroma 相关数据（视部署而定）
├── requirements.txt        # 根目录锁定版后端依赖（推荐与 venv 配合使用）
├── .env.example            # 环境变量模板
├── DEPLOY.md               # 分步部署（部分描述可能超前或滞后，以本文为准）
└── ARCHITECTURE.md         # 架构与优化记录
```

说明：

- 生产由 **单进程 FastAPI 托管**：`client/dist`（用户站）、`admin/dist`（管理站，路径 `/admin`）、`/data/images`（生成图）。  
- 存在 `admin-legacy/` 等历史目录时，**当前以后台 `admin/` 构建产物为准**。

## 五、环境变量（要点）

复制根目录 **`.env.example`**，在 **`server/.env`**（或与 `load_dotenv` 一致的路径）中填写实际值。常见项包括：

- **`ADMIN_PASSWORD`**：强密码；**不能留空，也不能为 `admin123`**，否则服务启动会失败。  
- **`MODEL_PROVIDER` / 各 LLM `*_API_KEY`**：至少一种可用模型。  
- **`XAI_API_KEY` 等**：图像能力；`MAX_CONCURRENT_IMAGES`、`SKIP_IMAGE_GEN` 等见 `.env.example`。  
- **`CORS_ORIGINS`**：生产必须改为真实前端/管理端域名（逗号分隔）。  
- **`DATABASE_URL`**：默认 SQLite；生产可换 `mysql+pymysql://...`。  
- **`MEMORY_ROOT`、`IMAGE_STORAGE_DIR`**：记忆与图片目录（与 `core/config`、挂载路径一致）。  
- **`PORT`**：Uvicorn 端口，默认 `8000`。

完整列表以 **`.env.example`** 为准。

## 六、本地开发

### 6.1 后端

```bash
cd /path/to/trandsai
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
# 配置 server/.env 后
cd server
python main.py             # 默认 0.0.0.0:8000，热重载
# 或: uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 6.2 用户前端

```bash
cd client
# 推荐使用 pnpm（package.json 含 overrides，避免与 React 18 类型冲突）
pnpm install
pnpm dev                   # 默认 http://localhost:5173，API 经 Vite 代理
```

环境变量 `API_BASE_URL`（如写在 `client/.env`）可改后端地址，与 `vite.config.ts` 中 `API_BASE` 一致。

### 6.3 管理前端

```bash
cd admin
npm install   # 或 pnpm install
npm run dev   # 默认 http://localhost:5174
```

### 6.4 开发时端口小结

| 服务 | 默认端口 |
|------|----------|
| FastAPI | 8000 |
| client (Vite) | 5173 |
| admin (Vite) | 5174 |

WebSocket 开发时经 client 代理路径 `/ws` 连到后端。

## 七、生产部署

### 7.1 构建前端

```bash
cd client && pnpm install && pnpm build
cd ../admin && npm install && npm run build
```

确保存在 **`client/dist`** 与 **`admin/dist`**（与 `server/core/config.py` 中 `DIST_DIR`、`ADMIN_DIR` 一致）。

### 7.2 运行后端

```bash
cd server
source ../venv/bin/activate
export PORT=8000   # 按需
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

- **一键对外入口**：仅暴露后端端口即可，由同一服务提供 API、用户 SPA、管理 SPA 与图片静态资源。  
- 前方可挂 **Nginx**：HTTPS、压缩、限流、长连接与 WebSocket 反代。  
- **CORS**、**数据库**、**密钥** 与 **数据目录持久化**（`data/`、`ai_companion.db` 或 MySQL、Chroma 目录）应在生产环境单独规划备份与权限。

### 7.3 与旧版 DEPLOY.md 的差异说明

- 管理端为 **React + Vite 构建**，不是旧文档中的纯静态 Alpine 方案。  
- 管理员密码以 **`main.py` 启动校验** 为准，示例中的弱密码不可用于启动。  
- 根目录 **`requirements.txt`** 为当前仓库锁定的依赖集合，安装时以该文件为主。

## 十、常见问题（精简）

1. **服务无法启动并提示密码不安全**：设置强 `ADMIN_PASSWORD`，且勿使用 `admin123`。  
2. **前端类型错误与 React 18**：用户端请用 **pnpm** 安装，避免 `@types/react` 主版本与 React 18 不一致。  
3. **生产环境图片或 API 变 HTML**：检查是否未构建 `dist`、Nginx 是否把 API 指到错误上游、或 SPA fallback 是否误伤 API（`main.py` 中已排除常见 API 与 `/admin` 等路径）。  
4. **WebSocket 断连**：确认代理支持 Upgrade、用户端是否在卸载时关闭连接。
